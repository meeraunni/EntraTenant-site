You don’t have permission to send messages from this mailbox (Outlook)

Usually one of three issues: wrong send permission, stale Outlook cache, or the wrong From address.

2026-03-03 7 min read Delegation
Fast truth: Full Access lets you open a mailbox. It does not let you send from it.

What the error means

Outlook tried to send using a mailbox address. Exchange rejected it because the sender does not have permission (or Outlook is using stale cached permission tokens).

Full Access vs Send As vs Send on Behalf

  • Full Access: open the mailbox and read it.
  • Send As: email appears from the mailbox.
  • Send on Behalf: email shows “User on behalf of Mailbox”.

User-side checks (quick)

  1. In the email, confirm the From address is the shared mailbox.
  2. Try sending from Outlook on the web. If OWA works, Outlook desktop is the problem.
  3. Close Outlook completely and reopen it.
  4. If still failing, create a new Outlook profile (fastest reliable fix).

Admin-side checks

  1. Confirm the user has Send As or Send on Behalf.
  2. Wait for replication after you assign permissions.
  3. If the mailbox was auto-mapped and permissions changed recently, Outlook may need a profile refresh.

PowerShell: verify permissions

# Connect-ExchangeOnline

$SharedMailbox = "shared@domain.com"
$User = "user@domain.com"

"Full Access"
Get-MailboxPermission -Identity $SharedMailbox |
  Where-Object { $_.User -like "*$User*" } |
  Format-Table User,AccessRights,IsInherited -Auto

"Send As"
Get-RecipientPermission -Identity $SharedMailbox |
  Where-Object { $_.Trustee -like "*$User*" } |
  Format-Table Trustee,AccessRights,IsInherited -Auto

"Send on Behalf"
(Get-Mailbox -Identity $SharedMailbox).GrantSendOnBehalfTo

PowerShell: assign the missing permission

Assign Send As

Add-RecipientPermission -Identity $SharedMailbox -Trustee $User -AccessRights SendAs -Confirm:$false

Assign Send on Behalf

Set-Mailbox -Identity $SharedMailbox -GrantSendOnBehalfTo @{Add=$User}

Troubleshooting matrix

  • OWA works, Outlook fails: cached tokens or profile. Recreate Outlook profile.
  • Both fail: permission missing or wrong From address.
  • Only some users fail: inconsistent delegation or multiple accounts causing From confusion.
