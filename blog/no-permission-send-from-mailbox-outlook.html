2026-03-02 Exchange Online Outlook Permissions

Fix: “You don’t have permission to send messages from this mailbox” (Outlook app)

This error usually shows up when Outlook tries to send as a mailbox you do not have the right permission for, or Outlook is holding onto stale permission state.

Understand the 3 permissions (most people mix them up)

  • Full Access: lets you open the mailbox. It does not grant sending permission.
  • Send As: email appears as if it came from the shared mailbox.
  • Send on Behalf: email shows “User on behalf of Shared Mailbox”.
Common trap: The user has Full Access but does not have Send As or Send on Behalf. Outlook lets them open the mailbox, then fails at send time.

Fast fix checklist (user side)

  • Confirm the From address is correct (the shared mailbox you expect).
  • Close Outlook fully and reopen it.
  • Sign out of Office apps and sign back in.
  • If it is Outlook desktop, try sending from Outlook on the web to compare behavior.
  • If it only fails in Outlook desktop, suspect cached auth or profile corruption.

Admin checklist (the real fix)

  • Confirm the user has Send As or Send on Behalf on that mailbox.
  • Wait for permission replication (it can take time).
  • If it still fails, force Outlook to refresh by removing and re-adding the mailbox or recreating the Outlook profile.

PowerShell: verify permissions

Replace values before running.

### Requires Exchange Online PowerShell
# Connect-ExchangeOnline

$SharedMailbox = "sharedmailbox@domain.com"
$User = "user@domain.com"

"Full Access:"
Get-MailboxPermission -Identity $SharedMailbox |
  Where-Object { $_.User -like "*$User*" -and $_.AccessRights -contains "FullAccess" } |
  Format-Table User, AccessRights, IsInherited -Auto

"Send As:"
Get-RecipientPermission -Identity $SharedMailbox |
  Where-Object { $_.Trustee -like "*$User*" -and $_.AccessRights -contains "SendAs" } |
  Format-Table Trustee, AccessRights, IsInherited -Auto

"Send on Behalf:"
(Get-Mailbox -Identity $SharedMailbox).GrantSendOnBehalfTo

PowerShell: assign Send As (if missing)

# Add Send As
Add-RecipientPermission -Identity $SharedMailbox -Trustee $User -AccessRights SendAs -Confirm:$false

PowerShell: assign Send on Behalf (if that is what you want)

# Add Send on Behalf
Set-Mailbox -Identity $SharedMailbox -GrantSendOnBehalfTo @{Add=$User}
Still failing? If permissions are correct but Outlook still throws the error, the next suspects are: cached tokens, profile corruption, automapping weirdness, or multiple accounts in the same profile causing From confusion.
