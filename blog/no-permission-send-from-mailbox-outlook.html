You don’t have permission to send messages from this mailbox

Most cases are one of three issues: missing Send As, missing Send on Behalf, or Outlook caching stale permissions.

2026-03-03 7 min read Mailbox delegation
Important: Full Access lets you open the mailbox. It does not grant sending rights.

What this error means

Outlook attempted to send using a mailbox address. Exchange rejected the send because the user does not have the correct send permission, or Outlook is still using a cached permission state.

Full Access vs Send As vs Send on Behalf

  • Full Access: open/read mailbox.
  • Send As: message appears from the mailbox.
  • Send on Behalf: message shows “User on behalf of Mailbox”.

User-side checks (fast)

  1. Confirm the From field is the shared mailbox (not the user mailbox).
  2. Try sending from Outlook on the web. If OWA works and Outlook desktop fails, it’s caching/profile.
  3. Close Outlook completely and reopen.
  4. If still failing, create a new Outlook profile (reliable reset).

Admin-side checks

  1. Confirm user has Send As or Send on Behalf (not just Full Access).
  2. Allow time for permission replication.
  3. If mailbox is auto-mapped and recently changed, Outlook may need a profile refresh.

PowerShell: verify permissions

# Connect-ExchangeOnline

$SharedMailbox = "shared@domain.com"
$User = "user@domain.com"

"Full Access"
Get-MailboxPermission -Identity $SharedMailbox |
  Where-Object { $_.User -like "*$User*" } |
  Format-Table User,AccessRights,IsInherited -Auto

"Send As"
Get-RecipientPermission -Identity $SharedMailbox |
  Where-Object { $_.Trustee -like "*$User*" } |
  Format-Table Trustee,AccessRights,IsInherited -Auto

"Send on Behalf"
(Get-Mailbox -Identity $SharedMailbox).GrantSendOnBehalfTo

PowerShell: assign the missing permission

Assign Send As

Add-RecipientPermission -Identity $SharedMailbox -Trustee $User -AccessRights SendAs -Confirm:$false

Assign Send on Behalf

Set-Mailbox -Identity $SharedMailbox -GrantSendOnBehalfTo @{Add=$User}

Verify and troubleshoot

  • OWA works, Outlook fails: Outlook cache/profile issue. New profile usually resolves.
  • Both fail: permission missing or wrong From address.
  • Only some users fail: inconsistent delegation or multiple accounts causing From confusion.
