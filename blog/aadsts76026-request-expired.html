AADSTS76026: The request has expired

Common SAML fix: set the Sign-on URL in the Enterprise App SAML configuration so My Apps launches create a fresh request.

2026-03-03 6 min read AADSTS
What the error means: Microsoft maps AADSTS76026 to RequestIssueTimeExpired, meaning the SAML request timestamp is expired. This is common when sign-in requests are replayed or started from the wrong entry point.

Typical symptoms

  • SP-initiated login (starting from the app) works.
  • IdP-initiated login (My Apps tile) fails with AADSTS76026.
  • Users retry and it “randomly” works sometimes (fresh vs stale request).

Why it happens (plain English)

SAML requests contain time-based values (IssueInstant / IssueTime). Entra validates that the request is fresh. If the request is stale, replayed, or generated from an outdated path, Entra rejects it as expired.

Main fix: set the Sign-on URL (Enterprise App SAML config)

If users launch the app from My Apps, Entra needs a correct entry point to start the app’s SAML flow. Set the Sign-on URL to the app’s SP-initiated login URL so launches generate a fresh SAML request.

  1. Open Entra admin center → Enterprise applications.
  2. Select your application.
  3. Go to Single sign-on → SAML.
  4. Under Basic SAML Configuration, click Edit.
  5. Set Sign-on URL to the app’s login/start URL (SP-initiated login URL).
  6. Save.
What should Sign-on URL be? Use the URL your application expects users to land on to start SAML login (the app’s own login page or SSO start endpoint).

Verify the fix

  1. Open an InPrivate/Incognito window.
  2. Go to myapps.microsoft.com.
  3. Click the application tile.
  4. Confirm you are redirected to the Sign-on URL and the login succeeds.

Other causes to check

  • Clock skew: if client or app servers are out of sync, the SAML request can appear expired.
  • Replay/caching: proxies or gateways caching auth pages can replay stale requests.
  • Bad redirect URI logic in the app: the app may be reusing an old request instead of creating a new one.

Admin notes

  • Use Entra Sign-in logs to confirm the failure stops after the change.
  • If only My Apps fails, start with Sign-on URL. If everything fails, check time sync and replay behavior first.
