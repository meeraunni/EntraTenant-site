Exchange Online · Shared Mailbox

Mails sent from a shared mailbox appear in personal Sent Items

This is common behavior. You can configure Exchange Online to keep a copy in the shared mailbox Sent Items folder.

2026-03-03 6 min read Sent Items

Why this happens

  • Users do not sign in as a shared mailbox directly. They sign in as themselves and access the mailbox via delegation.
  • By default, when a delegate sends as/on behalf, the sent copy can be stored in the delegate’s Sent Items.
Goal: Keep a copy in the shared mailbox Sent Items folder so the team has a single source of truth.

Fix: enable message copy settings

In Exchange Online PowerShell, enable both settings on the shared mailbox:

# Connect-ExchangeOnline

$SharedMailbox = "shared@domain.com"

Set-Mailbox -Identity $SharedMailbox `
  -MessageCopyForSentAsEnabled $true `
  -MessageCopyForSendOnBehalfEnabled $true

This instructs Exchange Online to save a copy of messages sent as the shared mailbox and sent on behalf of the shared mailbox in the shared mailbox Sent Items folder.

Verify the result

  1. Send a test email using Send As the shared mailbox.
  2. Confirm a copy appears in Shared mailbox → Sent Items.
  3. Repeat for Send on Behalf if your org uses it.

Client behavior notes

  • Some clients may still store a copy in the user’s Sent Items. That is normal. The important fix is: shared mailbox gets a copy.
  • New Outlook behavior can differ. Follow Microsoft’s troubleshooting guidance if you see inconsistent results.

Bulk enabling for multiple shared mailboxes

# Enable message copy settings for all shared mailboxes
Get-Mailbox -RecipientTypeDetails SharedMailbox -ResultSize Unlimited |
  Set-Mailbox -MessageCopyForSentAsEnabled $true -MessageCopyForSendOnBehalfEnabled $true
Need help? If shared mailbox behavior differs across Outlook versions, we can standardize settings and document the expected workflow.
