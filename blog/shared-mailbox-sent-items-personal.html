Mails sent from a shared mailbox are appearing in personal “Sent Items”

This is one of the most common shared mailbox complaints because it looks “wrong” to users. The good news: it is usually expected behavior until you explicitly configure otherwise.

Why it happens

  • When a user sends as a shared mailbox, the client can save the sent copy in the user’s mailbox by default.
  • Different clients behave differently (Outlook desktop vs OWA vs mobile).
  • You can tell Exchange Online to keep a copy in the shared mailbox Sent Items folder.

The clean fix (Exchange Online settings)

Enable message copy settings so sent mail is stored in the shared mailbox Sent Items as well. This helps shared workflows and auditability. 

### Requires Exchange Online PowerShell
# Connect-ExchangeOnline

$SharedMailbox = "sharedmailbox@domain.com"

Set-Mailbox -Identity $SharedMailbox `
  -MessageCopyForSentAsEnabled $true `
  -MessageCopyForSendOnBehalfEnabled $true
What this does: It stores a copy of sent mail in the shared mailbox when users send as it or on behalf of it. Users may still have a copy in their personal Sent Items depending on client behavior and settings.

Outlook client side (when user experience matters)

  • In Outlook desktop, there are settings that influence where sent items are stored for shared mailboxes.
  • If you want one canonical “Sent Items” location for the team, enforce the Exchange mailbox copy settings above.

Verification checklist

  • Send a test message using Send As the shared mailbox.
  • Confirm a copy appears in Shared Mailbox → Sent Items.
  • If it does not appear, wait for replication and retest.
  • Confirm the user is actually sending from the shared mailbox, not their own address.

Common gotchas

  • Users confuse “From” and accidentally send as themselves.
  • Multiple accounts in one Outlook profile can cause From selection mistakes.
  • Mobile clients can behave differently than desktop Outlook.
