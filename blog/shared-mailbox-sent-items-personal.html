Shared mailbox sent mail appears in personal Sent Items

This is expected unless you explicitly enable mailbox-side sent item copying for delegated sends. Treat it as a configuration decision: what is your system of record for outbound mail from a shared identity?

2026-03-038 min readSent ItemsShared mailbox
What’s actually happening: the user is authenticated as themselves, sending under delegated rights. Without explicit mailbox-side copy settings, Exchange can store the sent copy in the delegate’s mailbox.

Why the Sent Items location diverges

Shared mailboxes aren’t “logged into” like a user mailbox. Users access them via delegation. When a user sends as the shared mailbox, Exchange has to decide where to store the sent item. Historically, many clients kept the sent item in the delegate mailbox. This is not a bug. It’s a default behavior that becomes a governance problem when teams expect the shared mailbox to be the system of record.

Engineering requirement: define the system of record

If multiple people use a shared mailbox identity (support@, billing@, info@), you need sent items in the shared mailbox for audit and operational continuity. If sent items are scattered across personal mailboxes, you lose visibility, continuity, and forensic traceability. You can still keep a personal copy, but the shared mailbox must receive a copy.

Remediation: enable mailbox-side sent item copy controls

Exchange Online provides mailbox properties that force a copy of sent messages to be saved in the shared mailbox Sent Items folder for delegated sends. Enable both Send As and Send on Behalf copy settings to cover both patterns. 

# Connect-ExchangeOnline

$SharedMailbox = "shared@domain.com"

Set-Mailbox -Identity $SharedMailbox `
  -MessageCopyForSentAsEnabled $true `
  -MessageCopyForSendOnBehalfEnabled $true

Verification (prove it’s working)

Verify using both patterns if your org uses both. Send a test email using Send As and confirm a copy lands in the shared mailbox Sent Items folder. Repeat for Send on Behalf if applicable. If users still see a personal copy, that can be client behavior. Your objective is the shared mailbox copy being present consistently.

Bulk enablement (standardize across shared mailboxes)

If this is policy, apply it consistently instead of handling mailbox-by-mailbox. Bulk enablement reduces drift and avoids “this shared mailbox works, that one doesn’t.” 

# Enable on all shared mailboxes
Get-Mailbox -RecipientTypeDetails SharedMailbox -ResultSize Unlimited |
  Set-Mailbox -MessageCopyForSentAsEnabled $true -MessageCopyForSendOnBehalfEnabled $true
