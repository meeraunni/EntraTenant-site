How to recall an email using Microsoft Defender

Outlook recall is inconsistent. Defender remediation is the practical way: find the message everywhere, take action, and verify the result.

2026-03-03 5 min read Email security
Quick definition: In Defender, “recall” means remediation. You locate the email and remove it (or move it) from user mailboxes.
Table of contents
  • Prerequisites
  • Step 1: Find the email
  • Step 2: Take action
  • Step 3: Verify
  • When this won’t help
  • Prevent repeats

Prerequisites

  • Access to Microsoft Defender portal.
  • Permission to search and remediate email (role depends on your tenant setup).
  • Message details: subject, sender, recipient, time window, or Message ID.

Step 1: Find the email

  1. Open the Defender portal.
  2. Go to Email & collaboration → Explorer (sometimes called Threat Explorer).
  3. Search using the best identifier you have:
    • Message ID (best)
    • Sender address
    • Subject
    • Recipient
  4. Confirm it is the exact message (timestamp + sender domain + subject).

Step 2: Take action

Pick the action based on the goal:

  • Move to Junk: low risk, reversible.
  • Soft delete: remove from mailbox, recoverable by admins.
  • Hard delete: aggressive removal. Use only when needed.
  1. Select the message(s) in Explorer.
  2. Click Take action.
  3. Choose the remediation action and submit.
  4. Document what you did (subject, sender, time, action, scope).

Step 3: Verify

  • Refresh Explorer results and check action status.
  • Spot check one recipient mailbox to confirm the message is gone or moved.
  • If users clicked links, treat as an incident: reset passwords, revoke sessions, investigate sign-ins.

When this won’t help

  • If the email was sent externally and users already copied the content elsewhere.
  • If the “damage” is already done (credential entry, malware execution).
  • If you don’t have the right licensing or permissions for remediation actions.

Prevent repeats

  • Tune anti-phishing and anti-spam policies.
  • Enable and enforce reporting (“Report message”).
  • Review Safe Links / Safe Attachments if available.
  • Run awareness training for common patterns (invoice, password reset, DocShare).
