How to recall an email using Microsoft Defender

Outlook recall is inconsistent. The modern approach is remediation: find the message in Explorer (Threat Explorer) and take action (move, soft delete, hard delete).

2026-03-03 5 min read Email remediation
Quick definition: In Defender, “recall” means you remediate delivered mail. You locate the message and remove it (or move it) from recipient mailboxes.

When to use Defender vs Outlook recall

  • Outlook recall: best-effort, client-dependent, not reliable for incident response.
  • Defender remediation: centralized, auditable, works even if users never opened the message.

Prerequisites

  • Access to Microsoft Defender portal (https://security.microsoft.com).
  • Defender for Office 365 features available in your tenant (Explorer / remediation actions vary by licensing).
  • At least one identifier: subject, sender, recipient, time window, or Message ID.

Step 1: Find the email in Explorer (Threat Explorer)

  1. Open Microsoft Defender portal.
  2. Go to Email & collaboration → Explorer.
  3. Search using the strongest signal you have:
    • Message ID (best)
    • Sender address / domain
    • Subject
    • Recipient
  4. Open the message details to confirm it’s the correct mail.

Step 2: Take action (move / soft delete / hard delete)

In Explorer, select the message(s), then choose Take action. Typical actions include:

  • Move to Junk / Inbox / Deleted Items (low risk, reversible)
  • Soft delete (removes from mailbox, still recoverable)
  • Hard delete (aggressive removal)
Tip: Name your remediation action clearly (example: “Invoice phishing – March 2026”) so you can track it later.

If actions require approval (AIR)

Some environments use Automated Investigation and Response (AIR), where actions can require approval before they execute. Review and approve pending actions in the Investigations workflow.

Step 3: Verify and document

  1. Refresh Explorer results and confirm action status.
  2. Spot-check one mailbox to confirm the message is removed/moved.
  3. Document:
    • Sender, subject, time window
    • Scope (how many recipients)
    • Action taken (soft delete, hard delete, move)
    • Any follow-up (password reset, session revoke, sign-in investigation)

Prevention: policies and alerting

  • Use Alert policies to notify admins when risky email activity occurs.
  • Use AIR workflows to speed response and reduce manual effort.
  • Harden anti-phishing and anti-spam settings based on patterns you see.
