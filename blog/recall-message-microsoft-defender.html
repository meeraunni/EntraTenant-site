MICROSOFT DEFENDER FOR OFFICE 365

Remove an email across mailboxes using Microsoft Defender

Use Explorer and remediation actions in Microsoft Defender for Office 365 to remove a delivered message. This is the practical alternative to “Outlook recall.”

2026-03-07 9 min read Defender Email remediation
← Back to Blog

If a message was delivered and you need it removed across mailboxes, use Microsoft Defender for Office 365 remediation. Outlook message recall is not reliable across clients, devices, and external recipients.

Applies to Microsoft Defender for Office 365 (Plan 1/2 features vary), Microsoft Defender portal, and Exchange Online.

Before you start

Collect at least one stable identifier for the message. Message ID is best. If you don’t have it, use sender + recipient + subject + a narrow time range. This reduces false positives when multiple similar messages exist.

Step 1: Find and scope the message in Explorer

In the Microsoft Defender portal, use Explorer (Threat Explorer) to locate the message and confirm delivery details. Focus on scope: who received it, where it landed (Inbox, Junk, Quarantine), and whether there were interactions such as URL clicks.

Tip Treat scope as part of the fix. If you can’t clearly identify the message and recipients, you can’t reliably remediate it.

Step 2: Run a remediation action

Once the correct message is identified, choose an action appropriate for the risk. For confirmed malicious content, remove it. If you are still validating, you can move to Junk or take a less destructive action while investigation continues.

Step 3: Review and approve actions (AIR)

If your tenant uses Automated Investigation and Response (AIR), some actions can be pending approval. In that case, remediation is not complete until the action is approved and the status indicates it finished successfully.

Verify the result

Verify in two places: 1) Defender action status (completed/failed), and 2) Explorer results for the message over the same time window. If the message is still present, do not retry randomly. Confirm you targeted the correct artifact and that the action applied to the intended recipients.

If the message is still present

Check whether you’re blocked by permissions, licensing, or scope. Also verify the message wasn’t delivered as multiple variants (for example, modified subjects or different attachments). If you suspect multiple variants, use tighter identifiers such as Internet Message ID or attachment hash where available.
