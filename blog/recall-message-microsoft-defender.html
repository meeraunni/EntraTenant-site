AADSTS76026: The request has expired

Fix AADSTS76026 for SAML apps that fail when launched from My Apps. Set the Sign-on URL so the app generates a fresh request, then verify in sign-in logs.

2026-03-07 8 min Entra ID SAML
Back to Blog

You might see AADSTS76026: The request has expired. Try to submit new request. when users launch a SAML app from My Apps, but the same app works when users start from the application’s own sign-in page.

Applies to Microsoft Entra ID enterprise applications configured for SAML single sign-on.

Why this error happens

In a SAML flow, the application generates a sign-in request that includes a timestamp. Entra validates that timestamp to reduce replay. If Entra receives an older request, it rejects it as expired.

This often occurs when the My Apps launch path does not send users to the right SSO entry point. Without a correct entry point, the app can reuse a stale request or fail to generate a fresh request at the time of launch.

Fix

Set the Sign-on URL in the enterprise app’s SAML configuration. This ensures My Apps sends users to the application’s SSO start URL so the app generates a new SAML request.

Steps

In the Entra admin center: go to Enterprise applications, select the app, open Single sign-on, select SAML, and edit Basic SAML Configuration. Set Sign-on URL to the application’s SSO start URL, then save.

If you do not know the SSO start URL, check the vendor documentation for “SP-initiated SSO URL” or “SSO start URL.”

Verify

Open an InPrivate/Incognito window, sign in to myapps.microsoft.com, and launch the app tile. Confirm the sign-in completes without AADSTS76026.

If the issue continues

If the Sign-on URL is correct and the error continues, focus on time consistency and request replay. Confirm app servers and any proxy/federation components have reliable time sync. Review gateway and WAF rules that might cache or replay authentication redirects.

Sign-in logs

Compare a failed My Apps launch with a successful app-initiated launch in Entra sign-in logs. This helps confirm the issue is launch-path specific.
