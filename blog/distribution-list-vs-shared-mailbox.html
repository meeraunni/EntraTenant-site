Distribution list vs shared mailbox

Choose the correct object based on how you want work to happen: broadcast delivery, or a shared queue with shared history.

2026-03-07 10 min read Exchange Online Mail objects
← Back to Blog

The quickest way to pick the wrong object is to ask “what can I create?” The better question is “what workflow do I need?” A distribution list and a shared mailbox solve different problems.

Decision summary Use a distribution list to deliver copies to individuals. Use a shared mailbox when multiple people need a shared inbox and shared history.

What a distribution list is designed to do

A distribution list (distribution group) delivers a copy of a message to each member. The message exists separately in each user mailbox. That model works for announcements and simple routing, but it does not create a shared queue.

What a shared mailbox is designed to do

A shared mailbox is a single mailbox that multiple users can access through delegation. Mail arrives in one place. This supports triage, ownership, and continuity. Sending also becomes a design choice: Send As (shared identity) or Send on Behalf (user attribution).

Governance considerations

Whichever object you choose, assign an owner and review access periodically. DL membership tends to expand silently. Shared mailbox permissions tend to expand quietly. Both create risk if there is no clear ownership.

Verify before you deploy

Before you put the address on a website or publish it externally, verify: who can send to it (internal vs external), who can read it, and how replies should appear. Most “mail problems” are really inconsistent expectations.
